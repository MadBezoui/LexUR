# Project State: LexUR (Leximax Universal-Regret)

**Last Updated:** June 21, 2026
**Current Status:** Finalized for EJOR Second Round Revision

This document provides a high-level overview of the essential components of the project, documenting the current state of the theoretical framework, the computational benchmark, and the manuscript following the successful global rename of the core method from LUR to **LexUR** and the resolution of the EJOR referee's revision requests.

## 1. Project Structure

The repository is structured into three main domains:

### `code/` (The Python Package)
The core Python implementation (`lexur` package) and computational testbed.
- **`lexur/methods.py` & `lexur/directopt.py`**: The algorithms. Implements discrete LexUR, its robust interval bounding layer, and the continuous exact LP/MILP sequentially formulated direct optimization solver. It also contains the implementations of the baseline methods (MMR, ASF, TOPSIS, SMAA, etc.).
- **`lexur/experiments.py` & `lexur/families.py`**: The benchmarking engine, featuring the broadened protocol that tests models against 10 held-out preference families and 8 Pareto front geometries.
- **`lexur/smartgrid.py`**: The domain-specific 14-generator stochastic economic dispatch case study.
- **`scripts/`**: Automation scripts to run the benchmark, verify numerical values (`check_manuscript_numbers.py`), and generate all `matplotlib` paper figures (`make_paper_figures.py`).
- **`tests/`**: A rigorous `pytest` suite ensuring absolute algorithmic correctness and, crucially, testing the empirical claims made in the manuscript (`test_manuscript_check.py`).

### `paper/` (The EJOR Manuscript)
The LaTeX source for the article submitted to the European Journal of Operational Research (EJOR).
- Broken into logical chunks in `paper/sections/` (`01_intro.tex`, `03_core.tex`, `04_direct.tex`, etc.).
- Contains the formal proofs of existence, strict Pareto compatibility (Theorem 4), tolerance-based perturbation bounds, and the exact interval-LexUR stability sets.
- Dynamically integrates results and figures generated by the `code/` scripts.

### `results/` (Artifacts and Outputs)
Cached computational results.
- **`tables/` & `protocol/`**: Raw benchmark CSVs and JSON manifests. (Recently updated to reflect the new `LexUR` nomenclature).
- **`figures/`**: The generated PDF figures that are compiled straight into the LaTeX paper.

## 2. Recent Major Milestones

The project recently underwent a major, complete refactoring cycle to address a borderline-reject/major-revision EJOR referee report. The following items have been fully resolved and integrated:

1. **Formalisation of Interval-LexUR**: The heuristic treatment of nadir-uncertainty was upgraded to a mathematically rigorous definition of "necessary" and "possible" interval winners over conservative bounds.
2. **Direct Continuous Solver Formalisation**: Vague optimization logic was replaced with an exact, sequential lexicographic minimax regret LP/MILP formulation.
3. **Calibrated Claims**: Language claiming that LexUR was the "best-performing group" was corrected to accurately state it is "highly competitive with" and "non-inferior" to standard robust baselines, specifically falling slightly outside the Nemenyi Critical Difference of MMR/ASF in the tail-loss evaluation.
4. **Global Renaming**: The core method was systematically renamed from **LUR** to **LexUR** across all source code, LaTeX text, test suites, internal caching CSVs, and generated PDFs.
